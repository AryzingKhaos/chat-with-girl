## 💡 关键洞察

### 1. **高分辨率图片** - 提到 Soul 的好处

**v2_with_soul 减少了"脑补"**：
```
❌ v2_original（不提Soul）:
   "男是男，我是男生的呢"  ← 这句话根本不存在！
   "你真是晕了？"          ← 应该是其他内容

✅ v2_with_soul（提到Soul）:
   对话干净，没有添加不存在的内容
```

**原因分析**：
- 提到 "Soul" 给了模型更多上下文
- 模型知道这是特定应用，更谨慎识别
- 减少了基于"常见对话"的推断

### 2. **低分辨率图片** - 不提 Soul 更好

```
✅ v2_original（不提Soul）:
   "我们是灵魂的一对喔"  ← 正确！

❌ v2_with_soul（提到Soul）:
   "我们是灵层的一对喔"  ← 错误！"灵魂" → "灵层"
```

**原因分析**：
- 低分辨率下，文字模糊
- 提到 "Soul"/"灵魂" 可能让模型过度关联
- 导致错误识别相似字

## 🏆 最终推荐方案

### 方案 A：统一使用 v2_with_soul（推荐）⭐⭐⭐⭐⭐

```typescript
这是一张 Soul 聊天应用的截图。请仔细识别图片中所有对话文字内容，包括时间、昵称和消息内容。

要求：
- 逐字识别每个中文字符
- 识别所有时间戳（如：11:47, 19:02）
- 识别表情符号（如：❤️）
- 忽略装饰性元素（如头像、背景、图标）
- 保持原有格式和换行
- 只输出实际可见的文字，不要添加任何解释

直接输出识别结果：
```

**理由**：
- ✅ Soul1 效果显著提升（减少脑补）
- ⚠️ Soul2 略有下降（但可接受）
- ✅ 整体收益 > 损失

**适用场景**：
- 主要处理高分辨率图片
- 愿意接受低分辨率图片的小幅准确度损失
- 简单统一的方案

---

### 方案 B：智能动态选择（最优但复杂）⭐⭐⭐⭐⭐

根据图片分辨率自动选择 prompt：

```typescript
// 在 route handler 中
const imageInfo = await getImageDimensions(imageBase64);

const prompt = imageInfo.width > 800 && imageInfo.height > 800
  ? `这是一张 Soul 聊天应用的截图...`  // 高分辨率：提到 Soul
  : `这是一张聊天软件的截图...`;       // 低分辨率：不提 Soul
```

**理由**：
- ✅ 高分辨率：提到 Soul（减少脑补）
- ✅ 低分辨率：不提 Soul（识别更准确）
- ✅ 两全其美

**缺点**：
- 需要额外代码检测图片尺寸
- 复杂度增加
