# 对话内容结构化展示

## 功能目标

将 OCR 识别返回的纯文本对话内容解析为结构化 JSON 数据,并分别展示聊天内容和非聊天内容。聊天内容以对话气泡形式呈现,便于用户确认和后续分析。

---

## 输入输出定义

### 输入

**来源**: `OcrContainer` 组件的 `ocrText` 状态(string 类型)

**格式**: OCR API 返回的纯文本,格式如下:

```
对方(昵称A): 消息内容1
自己(昵称B): 消息内容2
对方(昵称A): 消息内容3

--- 非对话信息 ---
系统提示文字
日期分隔线
其他非对话内容
```

**约束**:
- 对话内容格式: `[说话人(昵称)]: [消息内容]`
- 说话人标识: `对方(...)` 或 `自己(...)`
- 非对话信息分隔符: `--- 非对话信息 ---`
- 如果无法识别左右位置,说话人为 `未知`

---

### 中间数据结构

**类型定义路径**: `src/types/chat.ts`

#### MessageItem (单条消息)

```typescript
interface MessageItem {
  id: string;                    // 消息唯一标识,使用 UUID
  speaker: 'self' | 'other';     // 说话人角色
  nickname: string;              // 昵称(对方显示真实昵称,自己固定显示"自己")
  content: string;               // 消息内容
  timestamp?: number;            // 消息时间戳(未来扩展,当前 OCR 不提取)
}
```

#### ChatData (解析后的完整数据)

```typescript
interface ChatData {
  messages: MessageItem[];       // 对话消息列表
  metadata: string[];            // 非对话信息列表(如系统提示、日期等)
}
```

#### ParseResult (解析结果)

```typescript
interface ParseResult {
  success: boolean;              // 是否解析成功
  data: ChatData | null;         // 解析后的数据
  error: string | null;          // 错误信息
}
```

## 状态机 / 流程

### 解析流程

```
OCR 文本输入
    ↓
文本非空检查 ──No→ 返回空数据
    ↓ Yes
按行分割文本
    ↓
遍历每一行 ────────────┐
    ↓                  │
检查是否为空行? ─Yes─→ 跳过 │
    ↓ No               │
检查是否为分隔符?        │
    ↓ Yes              │
    标记进入非对话区     │
    ↓ No               │
已进入非对话区? ─Yes─→ 归类为非对话信息
    ↓ No               │
正则匹配对话格式         │
    ↓ Yes              │
    提取说话人+昵称      │
    提取消息内容         │
    生成 MessageItem    │
    ↓ No               │
    ⚠️ 直接归类为非对话信息  │
    （不再追加到前一条消息）  │
    ↓                  │
继续下一行 ←────────────┘
    ↓
返回 ChatData
```

**⚠️ 关于 else 分支的修正说明**：

旧版实现中，正则匹配失败的行会追加到前一条消息内容（`lastMessage.content += '\n' + trimmedLine`），这引入了两个 bug：
1. **"自己"消息被归为非对话信息**：对话开头的第一行就是"自己"的消息，且正则匹配失败（如 `自己: 内容` 无括号），此时 `messages.length === 0`，直接落入 `metadata`
2. **消息内容错位**：后续"自己"的消息格式不对时，被错误追加到前一条"对方"的消息内容里

新版应将所有正则匹配失败的行**直接归类为非对话信息**，不再追加到前一条消息。

## 约束条件

### 安全约束

- **XSS 防护**: 消息内容必须使用 React 默认的文本渲染,不使用 `dangerouslySetInnerHTML`
- **输入验证**: 解析前检查输入是否为空字符串,避免无效解析

### 性能约束

- **消息数量**: 单次展示消息数量建议不超过 **200 条**,超过时提示用户截图过长
- **解析耗时**: 解析函数应在 **100ms** 内完成(纯文本正则处理,无异步操作)
- **渲染性能**: 消息列表使用虚拟滚动(可选,当消息数量 > 100 时)

### 数据约束

- **昵称长度**: 最长 20 字符,超出部分截断并显示省略号
- **消息内容**: 保留原始换行符,支持多行文本
- **非对话信息**: 最多展示 50 条,超出部分折叠

### 样式约束

- **响应式布局**: 移动端(< 768px)气泡宽度 90%,桌面端 60%
- **可访问性**: 按钮支持键盘导航,气泡使用语义化 HTML
- **主题适配**: 颜色使用 Tailwind CSS 变量,支持暗黑模式

## 边界情况

### 1. 空输入

**场景**: `ocrText` 为空字符串或仅包含空白字符

**处理**: 返回空 `ChatData`,UI 显示"暂无对话内容"提示

---

### 2. 无有效对话

**场景**: 文本不包含任何符合 `[说话人(昵称)]: [内容]` 格式的行

**处理**:
- `messages` 为空数组
- 所有文本归入 `metadata`
- UI 显示警告:"未识别到有效对话,请检查截图内容"

---

### 3. 缺少非对话信息分隔符

**场景**: 文本中不包含 `--- 非对话信息 ---`

**处理**:
- `metadata` 为空数组
- 非对话信息区域隐藏或显示"无非对话信息"

---

### 4. 说话人为"未知"

**场景**: OCR 无法判断消息左右位置

**处理**:
- `speaker` 设置为 `'other'`(默认归类为对方)
- 气泡添加黄色边框警告样式
- 昵称显示为"未知"

---

### 5. 昵称包含特殊字符

**场景**: 昵称中包含 `:` 、`(` 、`)` 等特殊字符

**处理**:
- 正则表达式使用非贪婪匹配,提取第一个 `):` 之前的内容作为昵称
- 昵称中的特殊字符保留,不做转义(React 自动转义)

---

### 6. 消息内容为空

**场景**: 匹配到 `对方(昵称):` 但冒号后无内容

**处理**:
- `content` 为空字符串
- 气泡正常渲染,但显示为空(保留气泡结构)

---

### 7. 消息内容包含多行

**场景**: 单条消息内容包含换行符(OCR 识别多行消息)

**处理**:
- 保留换行符,使用 `whitespace-pre-wrap` 样式正确显示
- 气泡高度自适应内容

---

### 8. 对话数量超限

**场景**: 解析出超过 200 条消息

**处理**:
- 正常解析所有消息
- UI 顶部显示警告 Toast:"消息数量过多(N 条),可能影响性能"
- 建议用户分段上传截图

// 匹配对话格式: [说话人(昵称)]: [内容]
// ⚠️ 已修正：旧版正则 /^(对方|自己|未知)\((.+?)\):\s*(.*)$/ 存在以下缺陷：

const messageRegex = /^(对方|自己|未知)(?:[（(]([^）)]*)[）)])?:\s*(.*)$/;
