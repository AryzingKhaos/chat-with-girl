# chatWithGirl - AI 聊天助手

## 项目概述

这是一个基于 AI 的智能聊天助手项目，旨在帮助男性用户与女性进行更有效的文字沟通。系统通过分析聊天记录，理解对方的心理状态，并提供专业的回复建议。

## 用户场景

- **目标用户**：男性用户
- **使用场景**：与女性进行文字聊天
- **视角定位**：我方始终代表男性用户视角

## 核心功能

### 功能 1：获取聊天记录

系统支持两种方式获取用户的聊天记录：

#### 方式一：文字复制
- 用户直接复制粘贴聊天文字内容
- AI 自动识别对话中的"我"（男性用户）和"对方"（女性）
- 智能解析对话结构和发言顺序

#### 方式二：截图上传
- 用户上传聊天截图
- AI 通过图像识别技术解析截图内容
- 自动区分"我"和"对方"的发言
- 提取完整的对话文本

#### 数据存储
- 识别完成后，将聊天记录持久化存储到本地
- 保存对话历史，便于后续分析和回溯

### 功能 2：实时心理分析

系统对聊天记录进行实时智能分析：

- **对话历史分析**：综合分析整个聊天记录的上下文关系
- **情绪识别**：识别女性当前的情绪状态（开心、生气、冷淡、热情等）
- **心理状态评估**：分析女性当前的心理状态和需求
- **关系进展判断**：评估当前的关系阶段和互动质量
- **实时更新**：随着对话的进行，持续更新分析结果

### 功能 3：智能回复建议

基于分析结果，提供实用的回复建议：

- **多样化建议**：提供 3-5 条不同风格的回复选项
- **情境匹配**：根据当前对话情境和女性心理状态，定制化建议
- **风格选择**：
  - 幽默风趣型
  - 关心体贴型
  - 深度交流型
  - 轻松日常型
- **可行性评估**：每条建议都经过可行性评估，确保自然流畅
- **效果预测**：预测不同回复可能带来的反应

## 技术架构

### 前端技术栈

#### 核心框架
- **运行环境**：浏览器（Chrome/Edge/Firefox）
- **UI 框架**：React 18+ / Vue 3+ （待选型）
- **语言**：TypeScript 5+
- **构建工具**：Vite
- **包管理器**：pnpm

#### UI 组件库
- **组件库**：Ant Design / Element Plus / Shadcn UI（待选型）
- **样式方案**：
  - CSS Modules / Tailwind CSS（待选型）
  - 支持主题切换（明亮/暗黑模式）
- **图标库**：Iconify / Lucide

#### 状态管理
- **全局状态**：Zustand / Pinia（待选型）
- **服务端状态**：不涉及，纯本地应用
- **表单管理**：React Hook Form / VueUse

#### 路由管理
- React Router / Vue Router
- 支持多页面导航（聊天记录、分析结果、历史记录等）

#### 工具库
- **日期处理**：date-fns / dayjs
- **工具函数**：lodash-es
- **UUID 生成**：uuid（用于生成对话 ID）
- **文件处理**：file-saver（导出聊天记录）
- **图片处理**：browser-image-compression（截图压缩）

#### AI 集成
- **AI SDK**：直接调用 AI API（OpenAI / Claude / 国产大模型）
- **HTTP 客户端**：axios / ky
- **文本处理**：自然语言解析、情感分析
- **图像识别**：OCR 服务调用（如百度 OCR / 腾讯 OCR）

#### 开发工具
- **代码规范**：ESLint + Prettier
- **Git Hooks**：Husky + lint-staged
- **类型检查**：TypeScript strict mode
- **测试框架**：Vitest + Testing Library（可选）

### 后端

**本项目暂不涉及后端服务**，所有功能均在前端完成：
- 数据存储使用浏览器本地存储（localStorage + IndexedDB）
- AI 能力通过前端直接调用第三方 AI API
- 无需服务器部署，纯静态页面

### AI 能力

#### 文本识别
- **对话解析**：识别对话双方身份（"我" vs "对方"）
- **发言顺序**：解析时间线和对话流程
- **关键信息提取**：提取情绪关键词、话题转折点

#### 图像识别（截图解析）
- **OCR 识别**：调用第三方 OCR API 提取文字
- **布局分析**：识别左右对话框位置，判断发言方
- **文本结构化**：将识别结果转换为结构化对话数据
- **支持格式**：PNG、JPG、JPEG、WebP

#### 自然语言处理
- **情感分析**：识别每条消息的情感倾向（正面/中性/负面）
- **语义理解**：理解对话的真实意图和潜台词
- **关系阶段判断**：陌生期/熟悉期/暧昧期/稳定期

#### 心理分析模型
- **情绪识别**：开心、生气、失落、冷淡、热情、撒娇等
- **性格推测**：根据聊天风格推测性格特点
- **兴趣爱好**：分析话题偏好
- **互动质量**：评估对话的投入度和积极性

#### 建议生成模型
- **上下文感知**：基于完整对话历史生成建议
- **风格多样化**：生成不同风格的回复选项
- **可行性评分**：为每条建议评估成功概率
- **风险预警**：识别可能引起负面反应的回复

### 数据存储

本项目采用**纯前端存储方案**，无需后端数据库：

#### localStorage（轻量级数据）
- **用途**：存储用户配置、应用设置、简单状态
- **存储内容**：
  - 用户偏好设置（主题、语言等）
  - AI API 配置（API Key、模型选择）
  - 最近使用的功能记录
  - 临时会话状态
- **容量限制**：约 5-10MB
- **数据格式**：JSON 字符串

#### IndexedDB（大容量数据）
- **用途**：存储聊天记录、分析结果、历史数据
- **存储内容**：
  - 聊天对话完整记录（对话 ID、时间戳、消息列表）
  - AI 分析结果历史（情绪分析、心理评估）
  - 建议历史记录（生成的回复建议及使用情况）
  - 人物画像数据（如果支持多人管理）
  - 截图原图缓存（可选，用户可清理）
- **容量限制**：通常 50MB+，可扩展至数百 MB
- **数据库设计**：
  - `conversations` 表：对话主表
  - `messages` 表：消息详情
  - `analyses` 表：分析结果
  - `suggestions` 表：建议历史
  - `profiles` 表：人物画像

#### 数据管理
- **数据导出**：支持导出为 JSON 文件备份
- **数据导入**：支持从备份文件恢复
- **数据清理**：用户可手动清除历史数据
- **存储监控**：显示当前存储使用情况
- **隐私保护**：所有数据仅存储在用户本地浏览器，不上传云端

## 产品特点

1. **双模式输入**：支持文字和截图两种输入方式，灵活便捷
2. **智能识别**：自动区分对话双方，无需手动标注
3. **实时分析**：动态分析聊天进展和对方心理状态
4. **专业建议**：基于 AI 分析提供专业的聊天建议
5. **隐私保护**：聊天记录本地存储，保护用户隐私

## 注意事项

### 伦理准则
- 本项目旨在帮助用户提升沟通能力，而非操控他人
- 尊重女性，建议内容应真诚、尊重
- 保护用户隐私，聊天数据严格保密

### 技术限制
- AI 分析结果仅供参考，不能完全代替人类判断
- 建议需要用户根据实际情况调整使用
- 不同文化背景可能影响分析准确度
